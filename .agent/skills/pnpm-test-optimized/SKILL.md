# pnpm-test-optimized

## Description
Executes project tests or builds using local commands (`pnpm test` or `pnpm build`) and returns a highly condensed executive summary of the execution context. Use this skill whenever you need to verify changes, check if tests pass, or compile the project, to prevent loading heavy terminal outputs into active memory.

## Goal
Run continuous integration verification checks locally and evaluate the state using a tight, token-efficient summary.

## Instructions
1. When the user asks to "verificar los cambios", "correr tests", o "compilar/hacer build", DO NOT use generic terminal execution tools to run the raw commands.
2. Instead, invoke the local python script located in this skill directory: `python3 .agent/skills/pnpm-test-optimized/run_tests.py "pnpm test"` (or `"pnpm build"` depending on the context).
3. Read the output summary generated by the script:
   - If STATUS is SUCCESS: Conclude the execution and tell the user everything is green.
   - If STATUS is FAILED: Read only the lines provided under "LÍNEAS CLAVE DEL ERROR" to perform your diagnosis and fix the codebase. Do not attempt to read the full log file.

## Constraints
- Never execute raw `pnpm test` or `pnpm build` directly in the general terminal if this skill is available.
- Do not request the full log of a command unless the provided summary is completely ambiguous.
